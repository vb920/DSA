Part 1: Top 10 Low-Level Design (LLD) Problems
Focus for SDE-2: Working, compilable code, SOLID principles, Concurrency (locking), and Design Patterns (Strategy, State, Observer, Factory).

Design a Parking Lot: The baseline for entity mapping, inheritance, and calculating fees.

Design LRU Cache: A staple that bridges data structures and object-oriented design.

Design an Elevator System: Tests your ability to handle algorithms (SCAN) and complex state transitions.

Design a Vending Machine: The absolute best problem to demonstrate the State Design Pattern.

Design Movie Ticket Booking System: Crucial for proving you can handle thread safety and database locking (optimistic vs. pessimistic) to prevent double-booking.

Design Splitwise: Tests your ability to wrap a complex algorithm (debt simplification) in a clean API.

Design Tic Tac Toe: Great for practicing basic game loops and the Strategy Pattern for winning conditions.

Design a Chess Game: A heavy problem that tests deep inheritance, interface segregation, and movement validation.

Design an Online Shopping System (Amazon): Shows you can decouple a large system into smaller, manageable contexts (Cart, Inventory, Payment).

Design a Pub/Sub System: Tests your knowledge of system internals and the Observer Pattern.

Part 2: Top 10 High-Level Design (HLD) Problems
Focus for SDE-2: Scalability, CAP theorem trade-offs, database selection (SQL vs. NoSQL), caching, and identifying bottlenecks.

Design a URL Shortener (TinyURL): The baseline for API design, unique ID generation, and caching.

Design an E-commerce Checkout System: Tests how you handle distributed transactions, inventory locking, and eventual consistency.

Design an API Rate Limiter: A component design problem testing algorithms (Token Bucket) and distributed caching (Redis).

Design a Chat Application (WhatsApp): Tests real-time connections (WebSockets), message sequencing, and presence servers.

Design a Social Media Feed (Twitter): Tests your understanding of "Fan-out on Read" vs. "Fan-out on Write" patterns.

Design a Video Streaming Platform (Netflix): Tests your understanding of Content Delivery Networks (CDNs) and blob storage.

Design a Ride-Sharing App (Uber): Tests your knowledge of geospatial indexing (Geohashing/QuadTrees) to find nearby drivers.

Design an Ad Click Aggregator (Datadog): Tests your ability to handle massive write throughput using message queues (Kafka) and time-series databases.

Design a Cloud File Sync System (Google Drive): Tests block storage (chunking files) and offline conflict resolution.

Design a Flash Sale System (Ticketmaster): Tests macro-level concurrency—how to handle 100,000 users trying to buy the exact same item in the same second without melting the database.
